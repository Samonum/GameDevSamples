## Transformin components
We can create a tree of components, where each component is encapsulated by a parent component*. Components have a local scale, rotation and position, which are relative to their parent component, and a world scale/rotation/position which can be calculated using it's local scale/rotation/position and the parent's scale/rotation/position.

* Note that parent components are not parent classes! Parent classes are about inheritance, or an *is a* relation, parent components are about encapsulation, or a *has a* relation.

### 1. Create a GetWorldScale() method that calculates the scale of an object given its local scale and a parent object.

For example: If a component's parent component becomes twice as big, so should the current component.

### 2. Do the same for GetWorldRotation()

Rotation is a float that gives the amount of radians an object should be rotated

### 3. Do the same for GetWorldPosition()

The local position is relative not only to the parent object, but is also affected by the parent scale and rotation!
